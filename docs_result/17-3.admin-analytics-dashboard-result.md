## 2. Part A — Worker API

### 2.1 `apps/api/src/routes/admin.analytics.ts` (전체)

```typescript
import { Hono } from 'hono'
import type { AppEnv } from '../env'

const adminRouter = new Hono<AppEnv>()

adminRouter.get('/api/admin/analytics', async (c) => {
  // --- Auth ---
  const key = c.req.header('x-admin-key')
  if (!c.env.ADMIN_SECRET || key !== c.env.ADMIN_SECRET) {
    return c.json({ ok: false, error: 'Unauthorized' }, 401)
  }

  const db = c.env.DB

  // --- KPI counts ---
  const kpiRows = await db
    .prepare(
      `SELECT event_name, COUNT(*) as count
       FROM events
       WHERE event_name IN (
         'generate_start','generate_success','limit_exceeded',
         'result_view','upgrade_click'
       )
       GROUP BY event_name`,
    )
    .all<{ event_name: string; count: number }>()

  const kpi: Record<string, number> = {}
  for (const row of kpiRows.results) {
    kpi[row.event_name] = row.count
  }

  const starts    = kpi['generate_start']   ?? 0
  const successes = kpi['generate_success'] ?? 0
  const limits    = kpi['limit_exceeded']   ?? 0
  const views     = kpi['result_view']      ?? 0
  const upgrades  = kpi['upgrade_click']    ?? 0

  const pct = (num: number, den: number) =>
    den === 0 ? null : Math.round((num / den) * 1000) / 10

  // --- Plan distribution ---
  const planRows = await db
    .prepare(
      `SELECT user_state, COUNT(*) as count
       FROM events
       WHERE event_name = 'generate_start'
       GROUP BY user_state`,
    )
    .all<{ user_state: string; count: number }>()

  // --- Hourly (last 24 h) ---
  const hourlyRows = await db
    .prepare(
      `SELECT substr(created_at, 1, 13) as hour, COUNT(*) as count
       FROM events
       WHERE event_name = 'generate_start'
         AND created_at >= datetime('now', '-1 day')
       GROUP BY hour
       ORDER BY hour ASC`,
    )
    .all<{ hour: string; count: number }>()

  return c.json({
    ok: true,
    kpi: {
      generate_start:  starts,
      generate_success: successes,
      limit_exceeded:  limits,
      result_view:     views,
      upgrade_click:   upgrades,
      success_rate:    pct(successes, starts),
      limit_rate:      pct(limits, successes),
      upgrade_rate:    pct(upgrades, limits),
    },
    plan_distribution: planRows.results,
    hourly:            hourlyRows.results,
  })
})

export default adminRouter
```

### 2.2 인증 방식

| 동작 | 결과 |
|---|---|
| `x-admin-key` 헤더 없음 | 401 Unauthorized |
| 키가 `env.ADMIN_SECRET`과 불일치 | 401 Unauthorized |
| `env.ADMIN_SECRET` 미설정 | 401 Unauthorized (방어적 처리) |
| 키 일치 | 200, analytics JSON |

`ADMIN_SECRET`이 설정되지 않은 경우도 401로 처리하여 미설정 상태에서 endpoint가 열리는 것을 방지합니다.

### 2.4 rate 계산 공식

| 지표 | 계산 | 의미 |
|---|---|---|
| `success_rate` | `generate_success / generate_start × 100` | 시작 대비 완료율 |
| `limit_rate` | `limit_exceeded / generate_success × 100` | 완료 후 한도 초과율 |
| `upgrade_rate` | `upgrade_click / limit_exceeded × 100` | 한도 초과 후 업그레이드 클릭율 |

분모가 0이면 `null` 반환 (웹에서는 `—` 표시).

### 2.5 D1 쿼리 상세

**KPI 집계**
```sql
SELECT event_name, COUNT(*) as count
FROM events
WHERE event_name IN (
  'generate_start','generate_success','limit_exceeded',
  'result_view','upgrade_click'
)
GROUP BY event_name
```

**Plan Distribution**
```sql
SELECT user_state, COUNT(*) as count
FROM events
WHERE event_name = 'generate_start'
GROUP BY user_state
```

**Hourly (last 24h)**
```sql
SELECT substr(created_at, 1, 13) as hour, COUNT(*) as count
FROM events
WHERE event_name = 'generate_start'
  AND created_at >= datetime('now', '-1 day')
GROUP BY hour
ORDER BY hour ASC
```

`substr(created_at, 1, 13)` → ISO 8601에서 `"2026-03-02T10"` 형태로 시간 단위 집계.

## 3. Part B — Web Admin Page

### 3.1 `apps/web/src/app/admin/page.tsx` (전체)

```typescript
'use client'

import { useEffect, useState } from 'react'

const API_BASE = process.env.NEXT_PUBLIC_API_BASE ?? 'http://127.0.0.1:8787'
const ADMIN_KEY = process.env.NEXT_PUBLIC_ADMIN_KEY ?? ''

interface KPI {
  generate_start: number
  generate_success: number
  limit_exceeded: number
  result_view: number
  upgrade_click: number
  success_rate: number | null
  limit_rate: number | null
  upgrade_rate: number | null
}

interface PlanRow { user_state: string; count: number }
interface HourRow { hour: string; count: number }

interface AnalyticsData {
  kpi: KPI
  plan_distribution: PlanRow[]
  hourly: HourRow[]
}

export default function AdminPage() {
  const [data, setData] = useState<AnalyticsData | null>(null)
  const [error, setError] = useState<string | null>(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    fetch(`${API_BASE}/api/admin/analytics`, {
      headers: { 'x-admin-key': ADMIN_KEY },
    })
      .then(async (res) => {
        if (res.status === 401) throw new Error('Unauthorized — check NEXT_PUBLIC_ADMIN_KEY')
        if (!res.ok) throw new Error(`HTTP ${res.status}`)
        return res.json() as Promise<{ ok: boolean } & AnalyticsData>
      })
      .then((json) => setData(json))
      .catch((err: Error) => setError(err.message))
      .finally(() => setLoading(false))
  }, [])

  // ...렌더링
}
```

**데이터 흐름**: `useEffect` 마운트 시 한 번 fetch → `loading` → 오류 처리 → `data` 렌더링.

### 3.2 KPI 카드

8개 카드를 `auto-fit minmax(160px, 1fr)` 그리드로 배치합니다:

| 카드 | 값 |
|---|---|
| Total Generates | `kpi.generate_start` |
| Successes | `kpi.generate_success` |
| Success Rate | `kpi.success_rate`% |
| Limit Exceeded | `kpi.limit_exceeded` |
| Limit Rate | `kpi.limit_rate`% |
| Result Views | `kpi.result_view` |
| Upgrade Clicks | `kpi.upgrade_click` |
| Upgrade Rate | `kpi.upgrade_rate`% |

null인 rate는 `—`로 표시.

### 3.3 Plan Distribution 테이블

| 컬럼 | 내용 |
|---|---|
| User State | `anonymous` / `free` / `pro` |
| Count | 해당 플랜의 generate_start 수 |
| Share | `count / generate_start × 100`% (프론트에서 계산) |

### 3.4 Hourly 테이블

| 컬럼 | 내용 |
|---|---|
| Hour (UTC) | `"2026-03-02T10"` 형식, monospace |
| Count | 해당 시간대의 generate_start 수 |

데이터가 없으면 "No data yet." / "No data in the last 24 hours." 텍스트 표시.


## 7. 보안 고려 사항

- `ADMIN_SECRET`은 Worker 시크릿으로 관리 → 코드베이스에 노출되지 않음
- `NEXT_PUBLIC_ADMIN_KEY`는 브라우저에 노출됨 → 어드민 페이지 자체가 내부 전용임을 전제
- 추후 개선: IP 화이트리스트, 세션 기반 어드민 인증
- `ADMIN_SECRET` 미설정 시 항상 401 → 배포 미스 방어
