## 2. 보안 하드닝 상세

### 2.1 redirect_to 이중 검증

매직 링크 플로우에서 `redirect_to`는 두 곳에서 검증됩니다:

**① 요청 수신 시 (`POST /auth/request-magic-link`)**

```typescript
// apps/api/src/routes/auth.ts
const rawRedirect = typeof body.redirect_to === 'string' ? body.redirect_to : '/'
const redirect_to = rawRedirect.startsWith('/') ? rawRedirect : '/'
```

→ `/`로 시작하지 않는 값은 DB에 저장되기 전에 `/`로 치환됩니다.

**② 검증 시 (`GET /auth/verify`)**

```typescript
const redirectTo = c.req.query('redirect_to') ?? '/'
const safeRedirect = redirectTo.startsWith('/') ? redirectTo : '/'
return c.redirect(safeRedirect, 302)
```

→ URL에서 직접 조작된 `redirect_to` 파라미터도 차단됩니다.

**공격 시나리오 방어 예시**

| 입력값 | 처리 결과 |
|---|---|
| `/login/success` | 그대로 사용 |
| `https://evil.com` | `/`로 교체 |
| `//evil.com` | `/`로 교체 |
| `javascript:alert(1)` | `/`로 교체 |
| (미제공) | `/`로 기본값 |

### 2.2 매직 링크 TTL — 기존 구현 확인

`apps/api/src/services/auth.service.ts`:

```typescript
const expiresAt = new Date(Date.now() + tokenTtlMinutes * 60 * 1000).toISOString()

// linksRepo.create() 시 expires_at 저장
await linksRepo.create({ ..., expiresAt })
```

`verifyMagicLink()` 내부:

```typescript
if (link.used_at || link.invalidated_at || new Date(link.expires_at) < now) {
  return { ok: false, error: 'MAGIC_LINK_EXPIRED_OR_USED' }
}
```

기본 TTL: `MAGIC_LINK_TOKEN_TTL_MINUTES` 환경 변수, 미설정 시 15분.

| 오류 | 응답 코드 | 발생 조건 |
|---|---|---|
| `INVALID_TOKEN` | 400 | token_hash가 DB에 없음 |
| `MAGIC_LINK_EXPIRED_OR_USED` | 410 | 만료 / 사용됨 / 무효화됨 |

### 2.3 쿠키 이름 일관성 — 기존 구현 확인

| 파일 | 쿠키 이름 |
|---|---|
| `middleware/session-cookie.ts` | `aso_session` |
| `routes/auth.ts` (verify) | `aso_session` |
| `routes/auth.ts` (logout) | `aso_session` |

세 파일 모두 동일한 이름 사용 → 쿠키 불일치 버그 없음.

## 3. 데이터 무결성

### 3.1 email_normalized UNIQUE 인덱스 — migration 0002 확인

`apps/api/src/db/migrations/0002_revenue_v2_safe.sql`:

```sql
CREATE UNIQUE INDEX IF NOT EXISTS idx_user_auth_profiles_email_normalized
  ON user_auth_profiles(email_normalized)
  WHERE email_normalized IS NOT NULL;
```

`WHERE email_normalized IS NOT NULL` 조건부 인덱스이므로 익명 사용자(email 없음)는 제외하고, 이메일이 있는 경우에만 유니크 제약을 적용합니다.

**경쟁 조건 방어**: `verifyMagicLink()` 내부에서 `getByEmailNormalized()` 조회 후 `promoteToMember()` 호출 시 `ON CONFLICT(uid) DO UPDATE`를 사용하므로 동시 요청에도 안전합니다.

### 3.2 role 컬럼 — migration 0004 신규

#### `apps/api/src/db/migrations/0004_role_column.sql` (전체)

```sql
-- 0004_role_column.sql
-- Additive-only: adds role column to user_auth_profiles.
-- SQLite ALTER TABLE ADD COLUMN is safe when a DEFAULT is provided.
-- Existing rows will get role = 'user' automatically.

ALTER TABLE user_auth_profiles
  ADD COLUMN role TEXT NOT NULL DEFAULT 'user'
    CHECK(role IN ('user', 'admin'));
```

**적용 특성**:

| 항목 | 내용 |
|---|---|
| 방식 | SQLite `ALTER TABLE ADD COLUMN` |
| 안전성 | DEFAULT 값이 있으므로 라이브 DB에 즉시 적용 가능 |
| 기존 행 | 자동으로 `role = 'user'` 할당됨 |
| 허용값 | `'user'` 또는 `'admin'` — CHECK 제약 |
| 어드민 지정 | SQL UPDATE로만 가능 (API 경로 없음) |

#### `apps/api/src/repositories/user-auth-profiles.repo.ts` 인터페이스 업데이트

```typescript
export interface UserAuthProfile {
  uid: string
  email: string | null
  email_normalized: string | null
  user_type: 'anonymous' | 'member'
  role: 'user' | 'admin'           // ← 추가됨
  email_verified_at: string | null
  created_at: string
  updated_at: string
}
```

## 4. /dashboard 페이지

### 4.1 `apps/web/src/app/dashboard/page.tsx` (전체)

```tsx
'use client'

import { useEffect, useState } from 'react'
import Link from 'next/link'
import TopNav from '@/components/TopNav'

const API_BASE = process.env.NEXT_PUBLIC_API_BASE ?? 'http://127.0.0.1:8787'

interface MeData {
  uid: string
  authenticated: boolean
  email: string | null
  plan: 'anonymous' | 'free' | 'pro'
  usage: {
    policy: string
    limit: number | null
    used: number
    remaining: number | null
    reset_at: string | null
  }
}

const PLAN_LABEL: Record<string, string> = {
  anonymous: 'Free (anonymous)',
  free: 'Free',
  pro: 'Pro',
}

const PLAN_COLOR: Record<string, string> = {
  anonymous: '#6b7280',
  free: '#2563eb',
  pro: '#7c3aed',
}

const cardStyle: React.CSSProperties = {
  border: '1px solid #e5e7eb',
  borderRadius: 12,
  padding: '20px 24px',
  marginBottom: 16,
}

const labelStyle: React.CSSProperties = {
  fontSize: 11,
  fontWeight: 700,
  color: '#9ca3af',
  letterSpacing: '0.08em',
  marginBottom: 6,
}

export default function DashboardPage() {
  const [data, setData] = useState<MeData | null>(null)
  const [loading, setLoading] = useState(true)
  const [fetchError, setFetchError] = useState<string | null>(null)

  useEffect(() => {
    fetch(`${API_BASE}/api/me`, { credentials: 'include', cache: 'no-store' })
      .then((r) => {
        if (!r.ok) throw new Error(`HTTP ${r.status}`)
        return r.json() as Promise<MeData>
      })
      .then(setData)
      .catch((e: unknown) => setFetchError(e instanceof Error ? e.message : 'Network error'))
      .finally(() => setLoading(false))
  }, [])

  if (loading) {
    return (
      <main style={{ fontFamily: 'sans-serif' }}>
        <TopNav />
        <div style={{ maxWidth: 480, margin: '80px auto', padding: '0 20px', textAlign: 'center', color: '#6b7280' }}>
          Loading…
        </div>
      </main>
    )
  }

  if (fetchError) {
    return (
      <main style={{ fontFamily: 'sans-serif' }}>
        <TopNav />
        <div style={{ maxWidth: 480, margin: '80px auto', padding: '0 20px', textAlign: 'center' }}>
          <p style={{ color: '#dc2626', fontSize: 14 }}>Failed to load dashboard: {fetchError}</p>
        </div>
      </main>
    )
  }

  if (!data?.authenticated) {
    return (
      <main style={{ fontFamily: 'sans-serif', color: '#111827' }}>
        <TopNav />
        <section style={{ maxWidth: 480, margin: '80px auto', padding: '0 20px', textAlign: 'center' }}>
          <div style={{ fontSize: 48, marginBottom: 16 }}>🔒</div>
          <h1 style={{ fontSize: 26, marginBottom: 12 }}>Login required</h1>
          <p style={{ color: '#6b7280', marginBottom: 32 }}>
            Sign in to view your plan and usage.
          </p>
          <Link
            href="/login"
            style={{
              display: 'inline-block',
              padding: '10px 28px',
              background: '#2563eb',
              color: '#fff',
              borderRadius: 6,
              fontWeight: 700,
              textDecoration: 'none',
            }}
          >
            Go to Login
          </Link>
        </section>
      </main>
    )
  }

  const { usage } = data
  const planLabel = PLAN_LABEL[data.plan] ?? data.plan
  const planColor = PLAN_COLOR[data.plan] ?? '#111827'

  return (
    <main style={{ fontFamily: 'sans-serif', color: '#111827' }}>
      <TopNav />

      <section style={{ maxWidth: 480, margin: '56px auto 80px', padding: '0 20px' }}>
        <h1 style={{ fontSize: 28, marginBottom: 4 }}>Dashboard</h1>
        {data.email && (
          <p style={{ color: '#6b7280', fontSize: 14, marginBottom: 32 }}>{data.email}</p>
        )}

        {/* 플랜 카드 */}
        <div style={cardStyle}>
          <div style={labelStyle}>PLAN</div>
          <div style={{ fontSize: 22, fontWeight: 800, color: planColor }}>{planLabel}</div>
          {data.plan !== 'pro' && (
            <Link
              href="/pricing"
              style={{ display: 'inline-block', marginTop: 10, fontSize: 13, color: '#2563eb', textDecoration: 'none', fontWeight: 600 }}
            >
              Upgrade to Pro →
            </Link>
          )}
        </div>

        {/* 사용량 카드 */}
        <div style={{ ...cardStyle, marginBottom: 32 }}>
          <div style={labelStyle}>USAGE</div>
          {usage.policy === 'unlimited' ? (
            <div style={{ fontSize: 20, fontWeight: 700, color: '#059669' }}>Unlimited</div>
          ) : (
            <>
              <div style={{ fontSize: 20, fontWeight: 700 }}>
                {usage.used} / {usage.limit ?? '∞'}
                <span style={{ fontSize: 14, fontWeight: 400, color: '#6b7280', marginLeft: 8 }}>
                  used
                </span>
              </div>
              {usage.remaining !== null && (
                <div style={{ fontSize: 14, color: '#6b7280', marginTop: 4 }}>
                  {usage.remaining} remaining
                  {usage.reset_at &&
                    ` · resets ${new Date(usage.reset_at).toLocaleDateString(undefined, { month: 'short', day: 'numeric' })}`}
                </div>
              )}
            </>
          )}
        </div>

        {/* 주요 CTA */}
        <Link
          href="/try"
          style={{
            display: 'block',
            padding: '13px 24px',
            background: '#2563eb',
            color: '#fff',
            borderRadius: 8,
            fontWeight: 700,
            textDecoration: 'none',
            textAlign: 'center',
            fontSize: 16,
          }}
        >
          Go to /try →
        </Link>
      </section>
    </main>
  )
}
```

### 4.3 `/api/me` 응답 매핑

```typescript
// GET /api/me 응답 구조
{
  uid: string
  authenticated: boolean      // ← 인증 분기 기준

  plan: 'anonymous'|'free'|'pro'  // ← 플랜 카드

    policy: 'lifetime'|'monthly'|'unlimited'
```

플랜별 `usage.policy` 대응:

| 플랜 | policy | limit |
|---|---|---|

| `free` | `monthly` | 5회/월 |
| `pro` | `unlimited` | null |

### `user_auth_profiles` 최종 스키마

```sql
CREATE TABLE user_auth_profiles (
  uid                TEXT PRIMARY KEY REFERENCES users(uid) ON DELETE CASCADE,
  email              TEXT,
  email_normalized   TEXT,
  user_type          TEXT NOT NULL DEFAULT 'anonymous'
                       CHECK(user_type IN ('anonymous','member')),
  role               TEXT NOT NULL DEFAULT 'user'            -- migration 0004
                       CHECK(role IN ('user', 'admin')),
  email_verified_at  TEXT,
  created_at         TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%fZ','now')),
  updated_at         TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%fZ','now'))
);

-- migration 0002
CREATE UNIQUE INDEX idx_user_auth_profiles_email_normalized
  ON user_auth_profiles(email_normalized)
  WHERE email_normalized IS NOT NULL;
```

## 6. 마이그레이션 적용 방법

### 로컬

```bash
cd apps/api

# 로컬 D1에 새 마이그레이션 적용
npx wrangler d1 migrations apply aso-copilot --local

# role 컬럼 확인
npx wrangler d1 execute aso-copilot --local \
  --command "PRAGMA table_info(user_auth_profiles);"
# name=role, type=TEXT, notnull=1, dflt_value='user' 확인

# 로컬 테스트 계정을 admin으로 승격

  --command "UPDATE user_auth_profiles SET role='admin' WHERE email_normalized='you@example.com';"
```
