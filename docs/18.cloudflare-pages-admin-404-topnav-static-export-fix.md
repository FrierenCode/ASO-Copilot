# ASO Copilot: `/admin` 404 해결 + 글로벌 TopNav + 정적 Export 검증

작성일: 2026-03-03  
대상 프로젝트: ASO Copilot (`pnpm` 모노레포, `apps/web` Next.js static export, Cloudflare Pages)

## 1) 루트 원인 분석 경로 (if/then)

### A. `/admin` 정적 산출물 자체가 없는 경우
1. `apps/web/src/app/admin/page.tsx` 파일이 없거나, 정적 export 불가 형태(동적 세그먼트/서버 의존)면 `/admin`이 export되지 않음.
2. `pnpm -C apps/web build` 후 `apps/web/out/admin/index.html` (또는 `admin.html`)이 없으면 빌드 포함 실패.

### B. 산출물은 있는데 운영에서 404가 나는 경우
1. Cloudflare Pages 설정에서 Output Directory가 잘못되면 route 파일이 배포되지 않음.  
   - 올바른 경로: `apps/web/out`
2. `_routes.json`, `_redirects`, `_headers`가 `/admin` 라우팅을 막는 경우 404 가능.  
   - 이번 점검에서 해당 파일들은 앱 소스에 없음.
3. 정적 호스팅에서 `/admin` 접근 시 `admin.html` 파일형 라우팅이 매칭되지 않는 케이스가 있음.  
   - 대응: `trailingSlash: true` 설정으로 `/admin/index.html` 산출 강제.
4. `basePath`, `assetPrefix`, 링크 경로 불일치가 있으면 직접 접근만 404가 날 수 있음.  
   - 이번 수정에서는 basePath/assetPrefix 도입 없이 route 경로를 단순 유지.

## 2) 실제 코드 변경 내역 (파일별)

## 2-1. Export/라우팅 안정화
- `apps/web/next.config.ts`
  - `trailingSlash: true` 추가
  - 목적: 정적 산출물을 `/<route>/index.html` 형태로 생성해 Cloudflare Pages 경로 매칭 안정화

## 2-2. 글로벌 TopNav 추가
- `apps/web/src/components/TopNav.tsx` (신규)
  - 로고 텍스트: `ASO Copilot`
  - 링크: `Pricing`, `Login`, `Signup`
  - 조건부 `Admin` 링크: `NEXT_PUBLIC_ADMIN_KEY` 존재 시 노출
  - 언어 토글: KR/EN
- `apps/web/src/hooks/useLanguage.ts` (신규)
  - `localStorage` (`aso-copilot-language`) 기반 언어 저장/복원
  - `document.documentElement.lang` 동기화(`kr -> ko`, `en -> en`)

## 2-3. TopNav 적용 페이지
- `apps/web/src/app/page.tsx`
- `apps/web/src/app/try/page.tsx`
- `apps/web/src/app/result/page.tsx`
- `apps/web/src/app/admin/page.tsx`
- `apps/web/src/app/pricing/page.tsx`

위 5개 페이지에 공통 상단 내비게이션을 적용했고, 랜딩 페이지(`/`)는 `useLanguage`와 연동해 KR/EN 전환을 실제 카피에 반영.

## 2-4. `/login`, `/signup` 정적 페이지 추가
- `apps/web/src/app/login/page.tsx` (신규)
- `apps/web/src/app/signup/page.tsx` (신규)

MVP UI:
1. 이메일 입력
2. `Send magic link` 버튼
3. 백엔드 미구현 상태 안내(Coming soon 성격 메시지)

둘 다 정적 export 가능한 클라이언트 페이지로 구성.

## 2-5. `/admin` 페이지 하드닝
- `apps/web/src/app/admin/page.tsx`
  - 클라이언트 fetch 유지: `https://api.gigigitg95.workers.dev/api/admin/analytics` (환경변수 우선)
  - `NEXT_PUBLIC_ADMIN_KEY` 누락 시 크래시 대신 안내 메시지 렌더링
  - 런타임-only 세그먼트 의존 없음(정적 라우트 유지)

## 3) 검증 명령 및 체크리스트

### 3-1. 로컬 빌드/산출물 확인
```powershell
pnpm -C apps/web build
Test-Path apps/web/out/admin/index.html
Test-Path apps/web/out/login/index.html
Test-Path apps/web/out/signup/index.html
```

기대 결과:
1. 빌드 성공
2. `True` 3개 확인

### 3-2. 로컬 정적 프리뷰
```powershell
pnpm exec wrangler pages dev apps/web/out --port 4173
```

브라우저 확인:
1. `http://localhost:4173/admin/`
2. `http://localhost:4173/login/`
3. `http://localhost:4173/signup/`
4. `http://localhost:4173/`, `/try/`, `/result/`, `/pricing/` 상단 TopNav 노출

### 3-3. 운영(Cloudflare Pages) 체크리스트
1. Build Command가 `apps/web` 빌드를 실제 수행하는지 확인  
   - 예: `pnpm -C apps/web build` 또는 루트에서 동일 결과를 내는 스크립트
2. Output Directory가 `apps/web/out`인지 확인
3. 배포 후 접근 확인:
   - `https://aso-copilot.pages.dev/admin`
   - `https://aso-copilot.pages.dev/login`
   - `https://aso-copilot.pages.dev/signup`
4. 404 미발생 확인
5. TopNav(Login/Signup/Pricing + KR/EN 토글) 페이지 공통 노출 확인
