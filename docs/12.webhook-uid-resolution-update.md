# Polar Webhook UID Resolution Update

대상 파일:
- `apps/api/src/routes/webhooks.polar.ts`

요청 요약:
1. 기존: `billingRepo.getUidByCustomerId` -> `customer.metadata.reference_id` fallback -> 실패 시 `uid_not_resolved`
2. 변경:
   - 1순위: `billingRepo.getUidByCustomerId(polarCustomerId)`
   - 2순위: `sub.metadata.reference_id`
   - 3순위: `customer.metadata.reference_id`
   - 없으면 `uid_not_resolved`
3. `subMeta` 추출 필수:
   - `const subMeta = sub.metadata as Record<string, unknown> | undefined`
4. entitlement 로직 변경 금지
5. uid 해석 블록만 수정

## 변경 적용 결과

- `subMeta` 추출 및 `subMeta?.reference_id` 사용 추가됨
- `customerMeta?.reference_id` fallback 유지됨
- 기존 `billing customer id -> uid` 매핑 조회 우선순위 유지됨
- `uid` 미해결 시 `uid_not_resolved` 실패 처리 유지됨
- entitlement 계산/업데이트 로직은 변경하지 않음

## Updated UID Resolution Section (Full)

```ts
// Resolve uid
const subMeta = sub.metadata as Record<string, unknown> | undefined
const customer = sub.customer as Record<string, unknown> | undefined
const customerMeta = customer?.metadata as Record<string, unknown> | undefined
const uidFromSubReferenceId = (subMeta?.reference_id as string) ?? null
const uidFromCustomerReferenceId = (customerMeta?.reference_id as string) ?? null

const billingRepo = new BillingRepo(c.env.DB)

// Try existing mapping first, then fall back to metadata
let uid = await billingRepo.getUidByCustomerId(polarCustomerId)
if (!uid && uidFromSubReferenceId) {
  uid = uidFromSubReferenceId
}
if (!uid && uidFromCustomerReferenceId) {
  uid = uidFromCustomerReferenceId
}

if (!uid) {
  await webhooksRepo.setFailed('polar', webhookId, 'uid_not_resolved')
  return c.json({ ok: false, error: 'Cannot resolve uid from event' }, 202)
}
```
