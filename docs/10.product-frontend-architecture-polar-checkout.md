## 1) Information Architecture (Routes / Components)

라우팅 구조 (App Router 기준):
1. `/` 기존 생성기 페이지
2. `/pricing` 요금제 비교 + 업그레이드 CTA
3. `/success` 결제 직후 상태 확인 페이지
4. `/pricing?checkout=cancelled` 취소 복귀 메시지 처리 (별도 `/cancel` 없이 운영 가능)

구성 컴포넌트 / 훅:
1. `src/hooks/useEntitlements.ts`
2. `src/app/pricing/page.tsx`
3. `src/app/success/page.tsx`
4. `src/components/LimitExceededPrompt.tsx` (모달/배너)
5. `src/components/PlanBadge.tsx` (선택, Free/Pro 표시 공통화)

상태 관리 원칙:
1. `uid`, `plan`은 `useEntitlements` 메모리 상태로만 유지
2. `localStorage` 기반 plan/entitlement 판단 금지
3. 서버 응답(`/v1/entitlements`)만 신뢰

## 2) Data Flow for Checkout URL Building

`/pricing` 페이지 기준:
1. 페이지 마운트 시 `GET /v1/entitlements` 호출
   - `credentials: 'include'`
   - `cache: 'no-store'`
2. API가 `uid`, `plan` 반환 (HttpOnly 서명 쿠키는 서버가 관리)
3. 프론트는 `uid`를 메모리 상태에만 보관
4. 체크아웃 URL 동적 생성:
   - `const checkout = new URL(POLAR_CHECKOUT_URL)`
   - `checkout.searchParams.set('reference_id', uid)`
5. 업그레이드 버튼 클릭 시:
   - `window.location.href = checkout.toString()`
6. `plan === 'pro'`이면 업그레이드 버튼 숨기고 `You're on Pro plan` 표시

핵심 보안 포인트:
1. plan 판단은 서버 결과만 사용
2. 클라이언트 저장소(localStorage/sessionStorage)에 entitlement 영속 저장 금지

## 3) Pricing / Success UX States

### `/pricing`
1. 로딩 상태:
   - Plan 표시: `Loading...`
   - Upgrade 버튼 비활성화
2. 성공 상태 (free):
   - Free vs Pro 비교
   - `Upgrade to Pro` 버튼 표시
3. 성공 상태 (pro):
   - 업그레이드 버튼 숨김
   - `You're on Pro plan` 표시
4. 오류 상태:
   - `Could not load entitlements. Please retry.`
   - `Retry` 버튼 제공

### `/success`
1. 쿼리에서 `checkout_id` 읽기 (`useSearchParams`)
2. 초기 메시지:
   - `Payment received. Activating your Pro access...`
3. entitlements 재조회
4. `plan === 'pro'`:
   - `Pro active` 확인 UI
   - 앱(대시보드) 복귀 링크
5. 아직 `free`:
   - `Payment processing can take a moment`
   - `Refresh status` 버튼으로 수동 재조회
6. 네트워크 오류:
   - 비차단 오류 메시지 + 재시도 버튼

## 4) Error Handling Integration with `/generate`

기존 생성 플로우를 깨지 않도록 429만 분기 처리:
1. `POST /generate` 호출 후 `res.status === 429`이면 JSON 파싱
2. `error === 'LIMIT_EXCEEDED'` 확인
3. 업그레이드 모달/배너 표시
4. CTA 클릭 시 `upgrade_url ?? '/pricing'`로 라우팅
5. 그 외 오류는 기존 오류 핸들링 유지

UX 카피(권장):
1. 타이틀: `Free limit reached`
2. 본문: `You've used 3 free generations this month.`
3. Primary CTA: `Upgrade to Pro`
4. Secondary: `Maybe later`

## 5) UX Copy (Short, Conversion-Oriented)

Pricing:
1. 헤드라인: `Unlock Unlimited ASO Generation`

3. Pro 라벨: `Unlimited generations + priority processing`
4. 버튼: `Upgrade to Pro`

1. 처리중: `Payment received. Activating your Pro access...`

4. 버튼: `Refresh status`
5. 링크: `Back to app`

Cancel (`/pricing?checkout=cancelled`):
1. `Checkout was canceled. You can upgrade anytime.`

## 6) Implementation Notes (Next.js on Cloudflare Pages)

환경변수 (`apps/web`):
1. `NEXT_PUBLIC_API_BASE=https://api.gigigitg95.workers.dev`
2. `NEXT_PUBLIC_POLAR_CHECKOUT_URL=https://buy.polar.sh/polar_cl_0fvC8twmmF1M3OcIl6wJ2JREHYcxS5Og0C6qb1ouo74`
3. Polar secret 등 민감정보는 절대 웹에 노출 금지

쿠키/CORS:
1. 프론트 fetch에 `credentials: 'include'` 필수
2. API는 `Access-Control-Allow-Credentials: true` 필요
3. API의 `Access-Control-Allow-Origin`은 명시적 origin 사용 (`*` 금지)
4. 가능하면 same-site 서브도메인(`app.example.com`, `api.example.com`) 운영 권장

캐시 정책:
1. 클라이언트: `cache: 'no-store'`
2. API 응답: `Cache-Control: no-store`
3. entitlements는 짧은 수명 메모리 상태로만 유지

복원력/운영성:
1. entitlements 실패 시 UI 재시도 버튼 제공
2. success 페이지는 수동 상태 갱신 지원
3. generate 메인 흐름은 유지, 429만 업그레이드 UX로 연결
4. 네트워크 오류는 사용자 액션 가능 상태(재시도/이동) 유지
