## 4) How uid Is Currently Resolved in Webhook Handler

현재 `uid` 해석 순서:
1. 이벤트 payload에서 `sub = event.data` 추출
2. `customer = sub.customer` 추출
3. `customerMeta = customer?.metadata` 추출
4. `uidFromMeta` 계산:
   - 우선순위 1: `customerMeta.reference_id`
   - 우선순위 2: `customerMeta.uid`
   - 없으면 `null`
5. DB 매핑 우선 조회:
   - `billingRepo.getUidByCustomerId(polarCustomerId)`
6. DB 매핑이 없고 `uidFromMeta`가 있으면 `uid = uidFromMeta`
7. 최종적으로 `uid`가 없으면:
   - `webhooksRepo.setFailed('polar', webhookId, 'uid_not_resolved')`
   - `202` 응답으로 처리 중단

즉, **고객 ID 매핑(DB) 우선 -> metadata fallback** 구조입니다.
