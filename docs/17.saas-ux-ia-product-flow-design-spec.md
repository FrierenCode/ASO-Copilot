## 1) Executive Summary (1-page)

### Product Direction
ASO-Copilot의 첫 경험을 “즉시 이해 + 즉시 체험 + 즉시 가치 확인” 구조로 재설계합니다.  
핵심은 첫 3초 내에 다음 3가지를 명확히 전달하는 것입니다.

1. 무엇을 하는가  
`앱 스토어 스크린샷 카피를 분석/개선해 Conversion Score와 개선안을 제공`

2. 무엇을 입력하는가  
`앱 이름 + 카테고리 + 스크린샷 문구`

3. 무엇을 얻는가  
`점수(Score) + 벤치마크 + 추천 + A/B/C 카피 + Export`

### Conversion Strategy
1. Anonymous 진입 장벽 최소화  
`로그인 없이 1회 Demo`를 랜딩에서 즉시 제공하여 가치 체감 우선.

2. 결과 직후 전환 유도  
Demo 결과 페이지의 1순위 CTA를 `회원가입/로그인`으로 고정.

3. 한도 기반 업그레이드 전환  
`남은 횟수 1회`에서 사전 경고, `한도 도달` 시 업그레이드 가치 제시.

4. 상태별 단일 CTA 정책  
각 상태별 primary CTA를 1개만 유지해 행동 분산을 방지.

### Why this reduces friction and improves conversion
1. 첫 화면 정보 구조를 “설명”이 아닌 “문제-입력-결과”로 압축해 인지 부하를 줄입니다.
2. 회원가입 전에도 결과를 보여주므로 신뢰 형성 후 전환이 가능합니다.
3. 결과 페이지에서 벤치마크를 제시해 점수의 의미를 즉시 이해하게 만듭니다.
4. Free/Pro의 가치 차이를 사용량 맥락에서 보여줘 업그레이드 의사결정을 빠르게 만듭니다.

### End-to-end Flows (A-D)
1. A) Landing -> Try Demo -> Result -> Signup -> Dashboard -> Run again  
2. B) Landing -> Signup -> Dashboard -> Create Project -> Run Analysis -> Result -> Save History -> Export  
3. C) Free warning(1 left) -> reached -> upgrade -> checkout -> return -> pro unlimited  
4. D) Returning -> login -> dashboard -> open history -> rerun optimization

## 6) Analytics Events Spec

### Common payload (all events)
```json
{
  "event_name": "string",
  "timestamp": "ISO8601",
  "route": "string",
  "locale": "ko-KR|en-US",
  "user_state": "anonymous|free|pro",
  "authenticated": true,
  "uid_hash": "string",
  "session_id": "string"
}
```

### Event table
| Event | Fire Point | Required payload fields |
|---|---|---|
| `generate_click` | Try/Dashboard에서 분석 버튼 클릭 시 | `project_id?`, `request_id`, `input_category` |
| `score_view` | Result 화면 최초 노출 시 | `run_id`, `score`, `category`, `benchmark_avg` |
| `limit_warning_view` | 남은 횟수 1일 때 경고 UI 노출 | `remaining=1`, `plan` |
| `limit_reached_view` | 429 후 게이트 노출 | `plan`, `upgrade_url` |
| `upgrade_view` | Pricing/Upgrade 블록 노출 | `current_plan`, `remaining` |
| `purchase_click` | Polar checkout 버튼 클릭 | `checkout_url`, `reference_id` |
| `purchase_success` | `/success`에서 Pro 확인 시 | `checkout_id`, `plan_after='pro'` |
| `signup_view` | Signup 페이지 노출 | `source` (`landing|result|gate`) |
| `signup_success` | Magic Link verify 성공 | `method='magic_link'` |
| `login_view` | Login 페이지 노출 | `source` |
| `login_success` | 로그인 완료 시 | `method='magic_link'` |
| `dashboard_view` | Dashboard 진입 시 | `project_count`, `recent_run_count` |
| `history_open` | 프로젝트 히스토리 열기 | `project_id`, `history_count` |
| `export_click` | Result/Project에서 export 클릭 | `run_id`, `format`, `plan` |

## 7) Data Model (D1) Minimal Schema (Projects/History)

### `projects`
- `project_id` TEXT PK
- `uid` TEXT NOT NULL
- `name` TEXT NOT NULL
- `category` TEXT
- `created_at` TEXT
- `updated_at` TEXT
- `archived_at` TEXT NULL

Indexes
- `(uid, updated_at DESC)`

### `project_runs`
- `run_id` TEXT PK
- `project_id` TEXT NULL (demo run 허용)
- `uid` TEXT NOT NULL
- `request_id` TEXT NOT NULL
- `input_json` TEXT NOT NULL
- `score` INTEGER NOT NULL
- `breakdown_json` TEXT NOT NULL
- `recommendation_json` TEXT NOT NULL
- `variants_json` TEXT NOT NULL
- `benchmark_avg` INTEGER NULL
- `source` TEXT (`demo|dashboard|rerun`)
- `created_at` TEXT NOT NULL

Indexes
- `(uid, created_at DESC)`
- `(project_id, created_at DESC)`
- `(uid, request_id)` UNIQUE (idempotency trace용)

### `run_exports` (optional but recommended)
- `export_id` TEXT PK
- `run_id` TEXT NOT NULL
- `uid` TEXT NOT NULL
- `format` TEXT (`txt|csv|json`)
- `created_at` TEXT

Indexes
- `(uid, created_at DESC)`

### Existing tables reused
- `user_entitlements` for Pro 판정
- `usage_monthly`, `usage_lifetime`, `generation_requests` for quota/idempotency

## 8) API Touchpoints (UI-level Contract)

| Endpoint | UI Usage | Notes |
|---|---|---|
| `GET /v1/entitlements` | plan/uid/limit 표시 | `cache: no-store`, `credentials: include` |
| `GET /api/me` | account menu, usage summary, dashboard header | auth 상태 + plan + remaining |
| `POST /generate` | demo/분석 실행 | `x-request-id` required |
| `POST /auth/request-magic-link` | signup/login form submit | email 입력 |
| `GET /auth/verify?token=` | 이메일 링크 완료 | session cookie set |
| `POST /auth/logout` | 계정 메뉴 로그아웃 | session invalidate |
| `GET /api/projects` | dashboard list | member/pro |
| `POST /api/projects` | 새 프로젝트 생성 | member/pro |
| `GET /api/projects/:id` | 프로젝트 상세 | member/pro |
| `GET /api/projects/:id/runs` | 히스토리 | member/pro |
| `POST /api/runs/:id/export` | export 실행 | free/pro 정책 반영 가능 |
| `GET /api/benchmarks?category=` | 결과 벤치마크 표시 | 캐시 가능(짧은 TTL) |

### `POST /generate` UI handling rules
1. `400 MISSING_IDEMPOTENCY_KEY` -> 클라이언트 request-id 생성 로직 점검 안내
2. `409 DUPLICATE_REQUEST` -> “이미 처리 중인 요청” 메시지
3. `429 LIMIT_EXCEEDED` -> 상태별 gate UI
4. `200` -> Result 페이지 이동 + `score_view` 이벤트 발화
